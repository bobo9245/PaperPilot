### 3. 방법

방법:
- TICoder의 전체 설계는 저장소 수준 코드 생성에서 계획 수립 단계와 재사용 단계로 나뉘는 흐름으로 이해된다. 계획은 구현 단계를 만들고, 재사용은 저장소 안의 관련 호출 대상 함수와 사용 패턴을 찾아 코드 생성에 활용한다.
- 테스트 기반 반복 계획 수립은 자연어 요구사항만 보는 기존 계획 방식과 다르다. 테스트 케이스를 기대 동작의 명세로 사용해 계획 결과를 반복적으로 조정함으로써, 모호하거나 불완전한 요구사항으로 인한 계획 오류를 줄이려 한다.
- 구현 인식 호출 대상 함수 검색은 이중 관점 유사도에 기반한다. 제공된 근거는 이 유사도가 기능적 측면과 구현 측면을 모두 포착한다고 설명하며, 단순 자연어 요구사항-코드 유사도 검색의 한계를 보완하려는 설계다.
- 사용 패턴 선택은 이중 단계로 구성된다. 먼저 구조 기반 군집화를 통해 후보 사용 패턴을 정리하고, 이후 퍼플렉시티 기반 필터링으로 더 관련성 높은 패턴을 고르는 방식으로 제시된다.
- 이 설계는 실제 개발자가 하나의 거의 동일한 코드 조각을 복사하기보다 여러 부분적으로 관련된 함수를 조합해 새 기능을 구현한다는 관찰에 기반한다. 따라서 함수 자체뿐 아니라 해당 함수가 어떻게 사용되는지도 중요한 단서로 본다.
